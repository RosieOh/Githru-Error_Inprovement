🔍 상세 문제점 분석

1. 타입/데이터 불일치로 인한 UI 오류

위치: packages/view/src/types/CommitMessageType.ts, packages/analysis-engine/src/types/CommitMessageType.ts, packages/vscode/src/types/CommitMessageType.ts
증상: 타입이 여러 곳에 중복 정의되어 동기화가 안 되면 UI/데이터 파싱 오류 발생. 각 패키지마다 다른 타입 정의로 인한 런타임 오류 가능성.

2. 외부 API(Gemini) 호출 실패 시 기능 미동작

위치: packages/analysis-engine/src/summary.ts (getSummary 함수)
증상: API 키가 없거나, 호출 실패 시 요약 미표시 및 안내 부족. 네트워크 오류나 API 제한 시에도 적절한 fallback 메시지가 없어 사용자가 혼란을 겪음.

3. 에러/예외 발생 시 사용자 안내 부족

위치: packages/vscode/src/extension.ts 등
증상: 에러 발생 시 사용자에게 친절한 안내 메시지가 부족, 문제 해결 방법 안내 미흡. 기술적인 에러 메시지가 그대로 노출되어 일반 사용자가 이해하기 어려움.

4. Octokit API Rate Limit/Abuse

위치: packages/analysis-engine/src/pluginOctokit.ts
증상: 대량 PR/커밋 fetch 시 API 제한, abuse 발생 가능. 사용자 안내/재시도 부족. Rate limit에 걸렸을 때 적절한 대응이 없어 기능이 완전히 중단됨.

🛠️ 개선 방안

1. 타입/데이터 불일치 문제 해결

개선 방안:

  • 공통 타입을 별도 패키지로 분리하여 중복 제거
  • 타입 정의를 단일 소스로 통합하고, 다른 패키지에서 import하여 사용
  • 타입 검증 로직 추가
  • 런타임 타입 체크 구현
// 새로운 공통 타입 패키지 생성: packages/types/src/CommitMessageType.ts
export enum CommitMessageType {
  FEAT = "feat",
  FIX = "fix",
  DOCS = "docs",
  STYLE = "style",
  REFACTOR = "refactor",
  TEST = "test",
  CHORE = "chore",
  UNKNOWN = "unknown"
}

export interface CommitMessageTypeConfig {
  type: CommitMessageType;
  description: string;
  emoji?: string;
}

// 타입 검증 함수
export function isValidCommitMessageType(type: string): type is CommitMessageType {
  return Object.values(CommitMessageType).includes(type as CommitMessageType);
}

// 런타임 타입 체크 및 변환
export function safeParseCommitMessageType(type: string): CommitMessageType {
  return isValidCommitMessageType(type) ? type as CommitMessageType : CommitMessageType.UNKNOWN;
}

2. 외부 API(Gemini) 호출 실패 시 기능 개선

개선 방안:

  • API 키가 없거나 호출 실패 시 사용자에게 안내 메시지 반환
  • 네트워크 오류 등 예외 상황에 대한 fallback 메시지 제공
  • 재시도 로직 구현
  • API 상태 모니터링
export async function getSummary(csmNodes: CommitRaw[]) {
  const commitMessages = csmNodes.map((csmNode) => csmNode.message.split('\n')[0]).join(', ');
  
  if (!apiKey) {
    // API 키가 없을 때 안내 메시지 반환
    return "[요약 불가] Gemini API 키가 설정되어 있지 않습니다. 설정에서 API 키를 추가해주세요.";
  }
  
  const maxRetries = 3;
  let lastError: Error | null = null;
  
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const response = await fetch(apiUrl + apiKey, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          contents: [{ parts: [{ text: `다음 커밋 메시지들을 요약해주세요: ${commitMessages}` }] }]
        })
      });
      
      if (!response.ok) {
        if (response.status === 429) {
          throw new Error("API 호출 한도를 초과했습니다. 잠시 후 다시 시도해주세요.");
        }
        throw new Error(`API 오류: ${response.status} ${response.statusText}`);
      }
      
      const data = await response.json();
      return data.candidates[0].content.parts[0].text.trim();
      
    } catch (error) {
      lastError = error as Error;
      console.warn(`Gemini API 호출 실패 (시도 ${attempt}/${maxRetries}):`, error);
      
      if (attempt < maxRetries) {
        // 지수 백오프로 재시도
        await new Promise(resolve => setTimeout(resolve, Math.pow(2, attempt) * 1000));
      }
    }
  }
  
  // 모든 재시도 실패 시 안내 메시지 반환
  return `[요약 불가] Gemini API 호출에 실패했습니다: ${lastError?.message || '알 수 없는 오류'}`;
}

3. 에러/예외 발생 시 사용자 안내 강화

개선 방안:

  • 사용자 친화적인 에러 메시지 시스템 구축
  • 에러 발생 시 해결 방법 안내
  • 에러 로깅 및 모니터링
  • 복구 가능한 에러와 치명적 에러 구분
// 에러 처리 유틸리티: packages/vscode/src/utils/errorHandler.ts
export class UserFriendlyError extends Error {
  constructor(
    message: string,
    public userMessage: string,
    public solution?: string,
    public isRecoverable: boolean = true
  ) {
    super(message);
    this.name = 'UserFriendlyError';
  }
}

export class ErrorHandler {
  static handle(error: unknown, context: string): void {
    if (error instanceof UserFriendlyError) {
      // 사용자 친화적 에러 메시지 표시
      vscode.window.showErrorMessage(
        error.userMessage,
        error.solution ? "해결 방법 보기" : undefined
      ).then((selection) => {
        if (selection === "해결 방법 보기" && error.solution) {
          vscode.window.showInformationMessage(error.solution);
        }
      });
    } else {
      // 일반 에러는 로깅 후 사용자에게 안내
      console.error(`[${context}] Error:`, error);
      vscode.window.showErrorMessage(
        "예상치 못한 오류가 발생했습니다. 개발자 도구를 확인하거나 재시작해주세요."
      );
    }
  }
  
  static createUserFriendlyError(
    originalError: unknown,
    context: string
  ): UserFriendlyError {
    const errorMessage = originalError instanceof Error ? originalError.message : "알 수 없는 오류";
    
    // 컨텍스트별 사용자 친화적 메시지 매핑
    const errorMappings: Record<string, { userMessage: string; solution?: string }> = {
      "git-not-found": {
        userMessage: "Git이 설치되어 있지 않습니다.",
        solution: "Git을 설치하고 PATH에 추가한 후 VSCode를 재시작해주세요."
      },
      "github-auth": {
        userMessage: "GitHub 인증이 필요합니다.",
        solution: "명령 팔레트에서 'Githru: Login with GitHub'를 실행해주세요."
      },
      "api-rate-limit": {
        userMessage: "API 호출 한도를 초과했습니다.",
        solution: "잠시 후 다시 시도하거나, 더 작은 저장소를 선택해주세요."
      }
    };
    
    const mapping = errorMappings[context] || {
      userMessage: "작업 중 오류가 발생했습니다.",
      solution: "VSCode를 재시작하거나 개발자 도구를 확인해주세요."
    };
    
    return new UserFriendlyError(errorMessage, mapping.userMessage, mapping.solution);
  }
}

4. Octokit API Rate Limit/Abuse 대응

개선 방안:

  • Rate limit 모니터링 및 대응
  • 재시도 로직 구현
  • 사용자에게 적절한 안내
  • API 호출 최적화
// Octokit API 래퍼 개선: packages/analysis-engine/src/pluginOctokit.ts
export class OctokitWrapper {
  private octokit: Octokit;
  private rateLimitInfo: { remaining: number; reset: number } | null = null;
  
  constructor(token: string) {
    this.octokit = new Octokit({ auth: token });
  }
  
  private async checkRateLimit(): Promise<void> {
    try {
      const response = await this.octokit.rest.rateLimit.get();
      this.rateLimitInfo = {
        remaining: response.data.rate.remaining,
        reset: response.data.rate.reset
      };
      
      if (this.rateLimitInfo.remaining < 10) {
        const resetTime = new Date(this.rateLimitInfo.reset * 1000);
        throw new UserFriendlyError(
          "Rate limit exceeded",
          "GitHub API 호출 한도에 도달했습니다.",
          `한도는 ${resetTime.toLocaleString()}에 재설정됩니다.`
        );
      }
    } catch (error) {
      if (error instanceof UserFriendlyError) {
        throw error;
      }
      console.warn("Rate limit 확인 실패:", error);
    }
  }
  
  private async retryWithBackoff<T>(
    operation: () => Promise<T>,
    maxRetries: number = 3
  ): Promise<T> {
    let lastError: unknown;
    
    for (let attempt = 1; attempt <= maxRetries; attempt++) {
      try {
        await this.checkRateLimit();
        return await operation();
      } catch (error) {
        lastError = error;
        
        if (error instanceof UserFriendlyError) {
          throw error;
        }
        
        if (attempt < maxRetries) {
          const delay = Math.pow(2, attempt) * 1000;
          console.log(`API 호출 실패, ${delay}ms 후 재시도 (${attempt}/${maxRetries})`);
          await new Promise(resolve => setTimeout(resolve, delay));
        }
      }
    }
    
    throw lastError;
  }
  
  async getPullRequests(owner: string, repo: string): Promise<any[]> {
    return await this.retryWithBackoff(async () => {
      const response = await this.octokit.rest.pulls.list({
        owner,
        repo,
        state: 'all',
        per_page: 100,
        sort: 'updated'
      });
      return response.data;
    });
  }
  
  async getCommits(owner: string, repo: string, sha?: string): Promise<any[]> {
    return await this.retryWithBackoff(async () => {
      const response = await this.octokit.rest.repos.listCommits({
        owner,
        repo,
        sha,
        per_page: 100
      });
      return response.data;
    });
  }
}
