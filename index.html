📊 문제점 요약

Githru VSCode Extension에서 발견된 주요 문제점 6가지를 우선순위별로 정리한 표입니다.

문제점 상세 위치 증상/설명 우선순위
Git 분석 데이터 로딩 문제 packages/vscode/src/webview-loader.ts, extension.ts Webview에 데이터 미전달, 화면 미표시 높음
GitHub 인증/토큰 문제 packages/vscode/src/extension.ts 인증 실패, 안내 부족 높음
외부 API(Gemini) 호출 실패 packages/analysis-engine/src/summary.ts 요약 미표시, 안내 부족 중간
StrictMode 비활성화 packages/view/src/index.common.tsx 잠재적 버그 미탐지 낮음

에러/예외 안내 부족 packages/vscode/src/extension.ts 등 사용자 불편, 문제 해결 어려움 높음

🔍 상세 문제점 분석

1. Git 분석 데이터가 제대로 로딩되지 않는 문제

위치: packages/vscode/src/webview-loader.ts (FIXME 주석), packages/vscode/src/extension.ts (fetchClusterNodes)
증상: Webview에 분석 데이터가 undefined로 전달되어 화면에 아무것도 안 나옴. 초기 데이터 로딩/동기화 문제 가능성.

2. GitHub 인증/토큰 문제

위치: packages/vscode/src/extension.ts (activate 함수)
증상: 인증이 안 되거나, 토큰이 없을 때 익스텐션이 정상 동작하지 않음. 안내 부족.

3. 외부 API(Gemini) 호출 실패 시 기능 미동작

위치: packages/analysis-engine/src/summary.ts (getSummary 함수)
증상: API 키가 없거나, 호출 실패 시 요약 미표시 및 안내 부족.

4. StrictMode 비활성화로 인한 잠재적 버그 미탐지

위치: packages/view/src/index.common.tsx
증상: React StrictMode가 꺼져 있어 잠재적 버그/비동기 문제를 미리 발견하지 못함.

5. Git 실행 환경 문제

위치: packages/vscode/src/utils/git.util.ts (findGitOnWin32, getRepo 등)
증상: Git 미설치, 경로 문제 등으로 익스텐션이 동작하지 않음. 에러 발생.

6. 에러/예외 발생 시 사용자 안내 부족

위치: packages/vscode/src/extension.ts 등
증상: 에러 발생 시 사용자에게 친절한 안내 메시지가 부족, 문제 해결 방법 안내 미흡.

🛠️ 개선 방안

1. Git 분석 데이터 로딩 문제 해결

개선 방안:

  • 분석 데이터가 undefined일 경우 사용자에게 명확한 안내 메시지를 표시하고, 재시도 버튼 등 fallback UI 제공
  • 데이터 fetch 함수에서 예외 상황을 명확히 처리하고, 실패 시 기본값 또는 에러 메시지 반환
// packages/vscode/src/webview-loader.ts
this._panel.webview.onDidReceiveMessage(async (message: { command: string; payload?: string }) => {
  try {
    // ... 기존 메시지 처리 ...
  } catch (e) {
    console.error("An error occurred while processing the webview message:", e);
    // 사용자에게 에러 메시지 전달
    this._panel.webview.postMessage({ 
      command: "showError", 
      payload: "분석 데이터 로딩에 실패했습니다. 새로고침 해주세요." 
    });
  }
});

// 분석 데이터가 undefined일 때 fallback 처리
if (!analyzedData) {
  this._panel.webview.postMessage({ 
    command: "showError", 
    payload: "분석 데이터가 없습니다. Git 저장소를 확인하세요." 
  });
}

2. GitHub 인증/토큰 문제 해결

개선 방안:

  • 토큰이 없을 때 사용자에게 인증 방법을 안내하는 메시지와 함께, 인증 버튼(명령어 실행) 제공
  • 인증 실패 시 재시도 로직 및 안내 강화
const githubToken: string | undefined = await getGithubToken(secrets);
if (!githubToken) {
  vscode.window.showErrorMessage(
    "GitHub 인증이 필요합니다. [여기서 로그인]을 클릭하세요.",
    "로그인"
  ).then((selection) => {
    if (selection === "로그인") {
      vscode.commands.executeCommand(COMMAND_LOGIN_WITH_GITHUB);
    }
  });
  throw new GithubTokenUndefinedError("GitHub 토큰이 없습니다. 인증을 진행해 주세요.");
}

3. 외부 API(Gemini) 호출 실패 시 기능 개선

개선 방안:

  • API 키가 없거나 호출 실패 시 사용자에게 안내 메시지 반환
  • 네트워크 오류 등 예외 상황에 대한 fallback 메시지 제공
export async function getSummary(csmNodes: CommitRaw[]) {
  const commitMessages = csmNodes.map((csmNode) => csmNode.message.split('\n')[0]).join(', ');
  if (!apiKey) {
    // API 키가 없을 때 안내 메시지 반환
    return "[요약 불가] Gemini API 키가 설정되어 있지 않습니다.";
  }
  try {
    const response = await fetch(apiUrl + apiKey, { ... });
    if (!response.ok) {
      throw new Error(`Error: ${response.status} ${response.statusText}`);
    }
    const data = await response.json();
    return data.candidates[0].content.parts[0].text.trim();
  } catch (error) {
    console.error("Error fetching summary:", error);
    // 네트워크/기타 오류 시 안내 메시지 반환
    return "[요약 불가] Gemini API 호출에 실패했습니다.";
  }
}

4. React StrictMode 활성화

개선 방안:

  • React StrictMode를 활성화하여 개발 단계에서 잠재적 문제를 조기에 발견
  • 성능 이슈가 있다면, 일시적으로만 비활성화하고 반드시 다시 활성화
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App";

export const initRender = () => {
  const rootContainer = document.getElementById("root") as HTMLElement;
  ReactDOM.createRoot(rootContainer).render(
    <React.StrictMode>
      <App />
    </React.StrictMode>
  );
};
// 주석: StrictMode를 활성화하면 useEffect 등에서 의도치 않은 두 번 호출이 있을 수 있으나, 
// 이는 개발 단계에서 버그를 조기에 발견하기 위함입니다.

5. Git 실행 환경 문제 해결

개선 방안:

  • Git 실행 파일을 찾지 못했을 때 사용자에게 명확한 안내 메시지 제공
  • Git 미설치 시 설치 안내 링크 제공
function findGitOnWin32() {
  return (
    findSystemGitWin32(process.env["ProgramW6432"])
      .then(undefined, () => findSystemGitWin32(process.env["ProgramFiles(x86)"]))
      .then(undefined, () => findSystemGitWin32(process.env["ProgramFiles"]))
      .then(undefined, () =>
        findSystemGitWin32(
          process.env["LocalAppData"] ? path.join(process.env["LocalAppData"]!, "Programs") : undefined
        )
      )
      .then(undefined, () => findGitWin32InPath())
      .catch(() => {
        vscode.window.showErrorMessage(
          "Git이 설치되어 있지 않거나 경로를 찾을 수 없습니다. [Git 설치하기](https://git-scm.com/download/win)"
        );
        throw new Error("Git 실행 파일을 찾을 수 없습니다.");
      })
  );
}

6. 에러/예외 발생 시 사용자 안내 개선

개선 방안:

  • 에러 타입별로 구체적인 해결 방법 안내
  • 사용자 친화적인 에러 메시지와 함께 해결 단계 제공
  • 에러 발생 시 로그 수집 및 분석 기능
// 에러 처리 유틸리티: packages/vscode/src/utils/errorHandler.ts
export enum ErrorType {
  GIT_NOT_FOUND = 'GIT_NOT_FOUND',
  GITHUB_AUTH_FAILED = 'GITHUB_AUTH_FAILED',
  NETWORK_ERROR = 'NETWORK_ERROR',
  API_RATE_LIMIT = 'API_RATE_LIMIT',
  DATA_PARSING_ERROR = 'DATA_PARSING_ERROR',
  UNKNOWN_ERROR = 'UNKNOWN_ERROR'
}

interface ErrorInfo {
  type: ErrorType;
  message: string;
  solution: string;
  action?: () => void;
}

const errorSolutions: Record<ErrorType, ErrorInfo> = {
  [ErrorType.GIT_NOT_FOUND]: {
    type: ErrorType.GIT_NOT_FOUND,
    message: "Git이 설치되어 있지 않습니다.",
    solution: "Git을 설치하고 환경 변수에 추가해주세요.",
    action: () => {
      vscode.env.openExternal(vscode.Uri.parse('https://git-scm.com/downloads'));
    }
  },
  [ErrorType.GITHUB_AUTH_FAILED]: {
    type: ErrorType.GITHUB_AUTH_FAILED,
    message: "GitHub 인증에 실패했습니다.",
    solution: "GitHub 토큰을 다시 설정해주세요.",
    action: () => {
      vscode.commands.executeCommand('githru.loginWithGithub');
    }
  },
  // ... 기타 에러 타입들 ...
};

export function handleError(error: Error, type: ErrorType = ErrorType.UNKNOWN_ERROR): void {
  const errorInfo = errorSolutions[type];
  
  const actions = ['다시 시도'];
  if (errorInfo.action) {
    actions.push('해결 방법 보기');
  }
  
  vscode.window.showErrorMessage(
    `${errorInfo.message} ${errorInfo.solution}`,
    ...actions
  ).then((selection) => {
    if (selection === '다시 시도') {
      // 재시도 로직
      vscode.commands.executeCommand('githru.refresh');
    } else if (selection === '해결 방법 보기' && errorInfo.action) {
      errorInfo.action();
    }
  });
  
  // 에러 로깅
  logger.error(`Error occurred: ${error.message}`, { type, stack: error.stack });
}
